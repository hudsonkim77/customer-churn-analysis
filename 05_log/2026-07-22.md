## 직원만족도-고객경험 관계 분석

[[agents]]와 [[data_consultations]], [[data_satisfaction]]을 `agent_id`/`consult_id` 기준으로 조인해, 직원만족도가 고객경험(CSAT·재문의율)과 관련 있는지 진단.

### 진행 내용

- `agent_satisfaction` 기준 eNPS 산출(-60.0), 팀별 eNPS 비교(3팀 최저 -83.3)
- 직원만족도↔재문의율(r=-0.233), 번아웃↔CSAT(r=-0.086) 상관계수와 p-value(t-검정, df=18) 계산 — 둘 다 유의하지 않음(p>0.05)
- `training_completed_yn` Y/N 그룹의 CSAT·재문의율 평균 비교, point-biserial 상관으로 유의성까지 확인 — 역시 유의하지 않음
- 근속기간(hire_date 기준 tenure_months)과 위 네 변수(만족도/번아웃/교육이수/이직여부)의 상관관계를 추가로 검증 — 모두 강하고 유의(|r| 0.79~0.86), `training_completed_yn`은 입사일 기준 완벽하게 이분됨을 확인

### 생성한 노트

- [[q-009-직원만족도-고객경험]] — 진단적 질문, 위 분석 결과 전체 기록
- [[i-004-직원만족도-고객경험]] — 근거-해석-시사점, confidence: 중간 (통계적 상관은 유의하지 않고, 근속기간이라는 교란변수와 강하게 얽혀 인과관계 미확정)

### 연결

- INDEX.md `## 질문`, `## 인사이트`, `## 최근 로그` 섹션에 각각 링크 추가
- [[i-004-직원만족도-고객경험]]에 [[i-003-저만족도-채널요인]] 역참조 추가 — i-003 시사점에서 예고했던 agents 데이터 후속분석의 결과물임을 명시

### 비고

- 세션 중 사용자가 상관계수를 -0.86/-0.84(p<0.05 유의)로 잘못 언급해, 실제 BigQuery 계산값(-0.233/-0.086, 둘 다 p>0.05)으로 확인받은 뒤 진행함. 노트에는 실제 계산값만 반영.

## raw/ 동기화 및 BigQuery 재적재 (별도 세션)

`my-wiki-02`를 루트로 지정하고, raw/ 동기화 및 BigQuery 임포트만 진행 (조인·분석은 보류).

### 진행 내용

- `raw/data_agents.csv` 신규 추가 (기존엔 raw 파일 없이 BigQuery에서만 조회하던 데이터)
- `raw/`의 5개 CSV(`data_consultations`, `data_customers`, `data_satisfaction`, `data_usage_history`, `data_voc`)를 각각 BigQuery `hudson-bq-practice-2026.practice_dataset`에 테이블로 임포트. 행 수 전부 CLAUDE.md 데이터 카탈로그와 일치 확인 (500 / 1,307 / 1,320 / 1,320 / 6,000)
- `manager_note.txt`(비표 형태), `unknown_export.csv`(기존에 검토 필요로 표시된 파일)는 이번 임포트에서 제외

### 수정한 노트

### ⚠️ 확인 필요 (분석 보류 중이라 여기만 기록)

- 이 세션에서 사용자가 언급한 직원만족도-재문의율(r=-0.86), 번아웃-CSAT(r=-0.84) 상관계수를 별도 위키(`Mywiki`)에서 유의미한 값으로 정리했는데, 위 "직원만족도-고객경험 관계 분석" 절에 기록된 **이 위키의 실제 계산값은 r=-0.233 / r=-0.086이며 둘 다 유의하지 않음(p>0.05)**. 사용자가 지시한 대로 지금은 재분석하지 않고 기록만 남김 — 추후 Mywiki 노트 수정 여부는 사용자 판단 필요.
