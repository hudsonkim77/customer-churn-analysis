- 위 두 차트 실행 중 **BigQuery `data_agents` 테이블이 이전 세션에서 재적재되어 값이 바뀐 사실을 발견** — [[i-004-직원만족도-고객경험]]을 현재 데이터 기준으로 전면 재계산·갱신함
- `charts/09_plotly_교육이수비교.py` 작성분을 실행 — 교육이수(`training_completed_yn`) Y/N 그룹별 CSAT 평균·재문의율 평균 막대 비교 차트. 결과값(CSAT 3.346 vs 3.284, 재문의율 21.15% vs 21.96%)이 [[q-009-직원만족도-고객경험]]·[[i-004-직원만족도-고객경험]]에 이미 기록된 수치와 정확히 일치 — `training_completed_yn` 컬럼은 `data_agents` 재적재 전후로 값이 바뀌지 않았다는 기존 추정을 재확인함
- Streamlit 대시보드(`app.py`)에 "상담원 관점: 직원만족도와 고객 경험" 섹션 신규 추가 — `charts/07`(eNPS 게이지)·`08`(번아웃-CSAT 산점도)·`09`(교육이수 비교) 로직을 이식하고, `st.selectbox`로 전체/1팀/2팀/3팀 필터링 가능하도록 구현. BigQuery 대신 [[agents|raw/data_agents.csv]]를 사용 — 이 raw 파일 값이 **현재 BigQuery `data_agents`와 일치함을 재검증**(아래 참고). `requirements.txt`에 `statsmodels` 추가(OLS 추세선용)
- 위 대시보드 기능을 Playwright로 로컬 실행(`streamlit run app.py`) 검증 — 팀 선택 드롭다운 전환 시 eNPS 게이지·번아웃CSAT 산점도·교육이수 비교 차트가 모두 재렌더링되고, 팀별 수치(1팀 eNPS -42.9·r=-0.82, 2팀 -14.3·r=-0.75, 3팀 -83.3·r=-0.93)가 [[i-004-직원만족도-고객경험]] 문서화 값과 정확히 일치함을 확인. 콘솔 에러 없음
- 커밋·push 후 Streamlit Cloud 실배포본(https://customer-churn-analysis-hudsonkim.streamlit.app/)도 헤드리스 브라우저로 확인 — 신규 섹션이 반영되어 있고 수치가 로컬과 동일함을 재확인 (Streamlit 앱이 iframe(`/~/+/`)에 렌더링되는 구조라 프레임 내부까지 접근해서 확인함)
