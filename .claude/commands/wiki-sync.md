raw/ 폴더에 새로 들어온 파일을 정리해서 위키에 반영한다. 아래 순서를 그대로 따른다.

1. `raw/` 폴더의 파일 목록과 `02_data/`, `00_inbox/`의 노트 목록을 **파일명 기준으로만** 대조한다. 이 단계에서는 파일 내용을 읽지 않는다.
2. 대조 결과 대응하는 노트가 없는 **새 파일만** 실제로 읽는다. 이미 노트가 있는 파일은 건드리지 않는다.
3. 새 파일의 내용에 따라 분기한다.
   - 표 형태(CSV, 스프레드시트, DB 테이블 등 컬럼 구조가 있는) 데이터면 `02_data/`에 스키마 노트를 생성한다. 컬럼명, 타입(추정 가능하면), 간단한 설명을 포함한다.
   - 표가 아닌 자유 형식 자료(메모, 문서, 텍스트 등)면 `00_inbox/`에 요약 노트를 생성한다.
4. 내용이 불분명해서 표 형태인지 자유 형식인지, 또는 어떻게 요약해야 할지 판단하기 어려우면 **추측하지 말고** `00_inbox/`에 "검토 요청" 메모를 남긴다. 무엇이 불분명한지, 어떤 판단이 필요한지 구체적으로 적는다.
5. 새로 만든 데이터 노트와 기존 `02_data/` 노트들 사이에 공통 컬럼이 있으면 위키링크(`[[노트명]]`)로 서로 연결한다.
6. `INDEX.md`에 이번에 새로 생긴 노트로 가는 링크를 해당 섹션(## 데이터 / ## 검토 필요)에 추가한다.
7. `05_log/오늘날짜.md` (예: `05_log/2026-07-10.md`) 파일에 이번에 처리한 내용을 기록한다. 없으면 새로 만든다. 처리한 raw 파일 목록, 생성한 노트, 남긴 검토 요청을 간단히 정리한다.

작업을 마치면 이번에 새로 처리한 파일과 생성/수정한 노트를 간단히 요약해서 보고한다.
